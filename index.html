Tecnolgías de web APIs

Permiten a dos componentes de software comunicarse mediante definiciones y protocolos.

Tecnologías de Web APIs

Una API (interfaz de programación de aplicaciones), es un conjunto de reglas definidas que permiten que diferentes aplicaciones se comuniquen entre sí. Actúa como una capa intermedia que procesa las transferencias de datos entre sistemas, permitiendo a las empresas abrir sus datos y funcionalidades de aplicaciones a desarrolladores externos, socios comerciales y departamentos internos dentro de sus empresas. (IBM, 2023)

Ejemplos de APIs:
  1. Las API de Google: Búsqueda, Gmail, Traductor o Maps.
  2. Paypal
  3. Incrustración de Videos de Youtube
  4. WhatsApp Business
  5. Instagram, Facebook

Las API pueden funcionar de maneras diferentes, según el momento y el motivo de su creación.

API de HTTPS

Una API de HTTPS (Protocolo Seguro de Transferencia de Hipertexto) es una interfaz de programación de aplicaciones que utiliza el protocolo HTTPS para permitir la comunicación segura entre clientes y servidores en la web. HTTPS es la versión segura de HTTP, donde los datos se cifran utilizando TLS (Transport Layer Security) o SSL (Secure Sockets Layer) para proteger la integridad y la confidencialidad de la información transmitida

Principales componentes y características de una API de HTTPS:
  • Protocolo seguro
    • Utiliza certificados SSL/TLS para cifrar los datos intercambiados entre el cliente (por ejemplo, un navegador web) y el servidor. Esto protege contra ataques de interceptación y otros tipos de espionaje de datos.
  • Autenticación
    • Aseguran que el servidor es auténtico y que la conexión es segura. El servidor presenta un certificado digital emitido por una autoridad certificadora confiable.
  • Cifrado
    • Protocolos que garantizan que los datos transmitidos no puedan ser leídos por terceros. TLS es la versión más moderna y segura en comparación con SSL.
  • Integridad de los datos
    • Aseguran que los datos no han sido alterados durante la transmisión. Si los datos han sido modificados, la suma de verificación o el hash serán diferentes, indicando corrupción o manipulación.
  • Encabezados HTTP
    • Incluye tokens de autenticación, como JWT (JSON Web Token), en los encabezados HTTP para identificar y autorizar usuarios.

    API de SOAP

    Una API de SOAP (Simple Object Access Protocol) es un protocolo basado en XML para intercambiar información estructurada en la implementación de servicios web en redes informáticas. Usa XML para sus mensajes y puede operar sobre varios protocolos de transporte, incluyendo HTTP, SMTP, y otros.

    Características de SOAP:
  • Protocolo basado en XMl
    • Todos los mensajes de SOAP se formatean en XML.
  • Transporte independiente
    • SOAP puede utilizar varios protocolos de transporte, como HTTP, SMTP, TCP, etc.
  • Estandares y extensibilidad
    • SOAP es un estándar del W3C y es extensible, permitiendo la incorporación de módulos adicionales de seguridad, transacciones, etc.
  • WS-Web Services Specifications
    • SOAP se integra con varias especificaciones WS- para características adicionales como seguridad (WS-Security), confiabilidad (WS-ReliableMessaging), y transacciones (WS-AtomicTransaction).

    API de REST

    Una API REST (Representational State Transfer) es un estilo de arquitectura de software que utiliza las operaciones estándar de HTTP para manipular recursos en una aplicación web. Las API REST son ampliamente utilizadas debido a su simplicidad y flexibilidad, permitiendo la comunicación entre clientes y servidores de manera eficiente.

    Características de REST:
  • Uso de HTTP
    • Las API RESTful utilizan los métodos HTTP estándar (GET, POST, PUT, DELETE, etc.) para operar con los recursos.
  • Recursos
    • Los recursos son entidades o datos que se exponen a través de la API, y se identifican mediante URLs.
  • Representaciones
    • Los recursos se representan normalmente en formatos como JSON o XML.
  • Stateless
    • Cada solicitud del cliente al servidor debe contener toda la información necesaria para entender y procesar la solicitud. El servidor no debe almacenar el estado del cliente entre solicitudes.
  • Cacheable
    • Las respuestas deben ser definidas como cacheables o no, para mejorar el rendimiento.
  • Cliente-servidor
    • El cliente y el servidor deben ser independientes y permitir una evolución separada.

    API de JWT

    JSON Web Tokens (JWT) son una manera compacta y segura de transmitir información entre partes como un objeto JSON. Son comúnmente utilizados en la autenticación y autorización de usuarios en aplicaciones web.

    Características de JWT:
  • Compacto
    • JWT se representa como una cadena de texto compacta, lo que facilita su uso en URLs, encabezados HTTP, y almacenamiento de datos.
  • Autocontenido
    • Un JWT puede contener toda la información necesaria para autenticar un usuario sin necesidad de buscar en una base de datos adicional.
  • Seguro
    • Los JWT pueden ser firmados utilizando un algoritmo de firma digital (como HMAC o RSA) para asegurar que los datos no han sido manipulados.


    Microservicios

    El software está compuesto por pequeños servicios independientes que se comunican a través de API.

    Microservicios

    Los microservicios (o arquitectura de microservicios) son un enfoque arquitectónico nativo de la nube en el que una sola aplicación se compone de muchos componentes o servicios más pequeños acoplados de forma flexible e independiente.

    Caracteristicas:

    1. Descomposición en servicios pequeños: En lugar de desarrollar una aplicación monolítica, los microservicios dividen la funcionalidad en servicios pequeños y autónomos, cada uno ejecutando un proceso específico

    2. Independencia y despliegue independiente: Cada microservicio es independiente y puede ser desarrollado, desplegado y escalado de manera independiente. Esto permite actualizaciones rápidas y flexibles sin afectar a otras partes del sistema

    3. Comunicación entre microservicios: Los microservicios se comunican entre sí generalmente a través de protocolos ligeros como HTTP o mensajería asincrónica. Esto facilita la integración y la evolución de la aplicación a medida que se agregan nuevos servicios.

    4. Escalabilidad y mantenibilidad: La arquitectura de microservicios facilita la escalabilidad horizontal, lo que significa que puedes escalar cada servicio individualmente según la demanda. Además, los equipos pueden trabajar de forma independiente en cada microservicio, lo que facilita el mantenimiento y la evolución continua del sistema

    5. Resiliencia y tolerancia a fallos: Al dividir la aplicación en servicios pequeños, los microservicios pueden aislar y manejar mejor los errores y fallos, lo que mejora la resiliencia general del sistema

    6. Gestión de datos: Cada microservicio puede tener su propia base de datos, lo que ayuda a mantener la independencia y la cohesión de los servicios. Sin embargo, esto también introduce desafíos en cuanto a la consistencia de los datos y la gestión de transacciones distribuidas

    7. Desafíos de implementación: Aunque los microservicios ofrecen numerosos beneficios, también plantean desafíos en áreas como la complejidad de la gestión de la arquitectura, la coordinación entre servicios y la monitorización de la salud del sistema

    En resumen, los microservicios son una arquitectura de software que ofrece flexibilidad, escalabilidad y mantenibilidad al dividir una aplicación en servicios independientes y autónomos. Sin embargo, su implementación requiere una planificación cuidadosa y la gestión de ciertos desafíos técnicos y organizativos.     Desarrollo de una web API

    Proporciona una interfaz de programación de apps (API) a través de la web, permitiendo que sistemas y plataformas se comuniquen

    Desarrollo de una web API


    El termino API que son las siglas de Application Programming Interface. Es un conjunto de reglas y definiciones que permite que diferentes aplicaciones se comuniquen entre sí. En términos simples, una API especifica cómo interactuar con un sistema de software, permitiendo que otras aplicaciones accedan a sus funciones, datos o servicios de manera controlada y segura.

    Proceso de desarrollo de una API

    1. Analisis de requerimientos: Identificar los requisitos funcionales y no funcionales de la API, incluidos los casos de uso y los datos que se manipularán.

    2. Planificación: Una vez que se han definido los requerimientos, se pasa a la planificación del proyecto, que incluye la definición de los objetivos del proyecto, la asignación de recursos, la estimación de tiempos y costos, y la definición de la metodología de desarrollo a utilizar. La planificación se basa en los requerimientos identificados.

    3. Diseño de la API: Definir los endpoints, métodos HTTP admitidos, parámetros de solicitud y respuesta, y esquema de autenticación.

    4. Implementación: Al desarrollar una API se debe seguir un diseño definido, utilizando herramientas y frameworks adecuados como por ejemplo Express.js para Node.js o Flask para Python.

    5. Pruebas: Realizar pruebas unitarias y de integración para garantizar que la API funcione según lo esperado y que maneje adecuadamente los errores

    6. Documentación: Crear documentación clara y detallada de la API, incluidos ejemplos de uso y casos de error.

    7. Despliegue: Implementar la API en un entorno de producción, asegurando que esté disponible y escalable.

    8. Montioreo y mantenimiento: Establecer un sistema de monitoreo para detectar problemas y realizar mantenimiento regular para garantizar que la API siga siendo eficiente y segura.

    9. Comercialización (opcional): Al desarrollar Apis que serán ofertadas al público o las empresas, entonces será conveniente publicar la API para monetizarla en algún mercado de APIs en donde los desarrolladores compran y vendan

    Recomendaciones

    1. Definir los objetivos de la API: Tener una comprensión clara de los problemas que intentará resolver y los usuarios a los que servirá.

    2. Seguir estandares y convenciones: Esto facilita la comprensión y el uso de la API por parte de otros desarrolladores, por ejemplo seguir los principios de REST, utilizar JSON o XML como formatos de intercambio de datos,etc.

    3. Documentar la API de manera exhaustiva: Es esencial para que otros desarrolladores puedan entender cómo usarla correctamente, se pueden utilizar herramientas como Swagger o Postman para generar documentación automáticamente

    4. Priorizar la seguridad: Se deben implementar medidas de seguridad como autenticación y autorización para proteger la API de accesos no autorizados.

    5. Versionar la API: A medida que la API evoluciona, es importante mantener la compatibilidad hacia atrás, versionar la API permite introducir cambios sin afectar a los usuarios existentes

    Una API web promueve la interoperabilidad entre sistemas, brinda flexibilidad para la creación de nuevas aplicaciones y servicios, facilita la escalabilidad de los sistemas, fomenta la reutilización de código y simplifica la integración con sistemas externos. En resumen, una API bien diseñada y desarrollada puede ser un activo invaluable para cualquier proyecto de software, permitiendo la construcción de aplicaciones robustas, ágiles e interoperables.

    Importancia y utilidad

  • Interconeción de sistemas: Permiten la comunicación y la integración entre diferentes sistemas y plataformas, lo que facilita la interoperabilidad y la colaboración entre aplicaciones.

  • Acceso a datos y funcionalidades: Proporcionan acceso a datos y funcionalidades específicas de una aplicación, lo que permite a otros desarrolladores construir sobre esa funcionalidad o crear nuevas aplicaciones que utilicen esos datos.

  • Construcción de ecosistemas: Las empresas pueden construir ecosistemas de aplicaciones alrededor de su API, fomentando la innovación y el desarrollo de nuevas soluciones por parte de terceros.

  • Escalabilidad: Las empresas pueden escalar su negocio permitiendo que múltiples clientes accedan a sus servicios de manera simultánea.

  • Facilita el desarrollo agil: Al separar la lógica del servidor y el cliente, las Web APIs permiten un desarrollo más ágil, donde equipos diferentes pueden trabajar de forma independiente en el frontend y el backend.


    • Manejo de contenedores

    Paquetes de software que incluyen todos los elementos necesarios para ejecutar tus productos en cualquier entorno

    Uso de Contenedores

    Los contenedores son paquetes ligeros que incluyen el código de las aplicaciones junto con sus dependencias, como versiones concretas de entornos de ejecución de ciertos lenguajes de programación y bibliotecas indispensables para ejecutar los servicios de software.


    Los contenedores constituyen un mecanismo de empaquetado lógico en el que las aplicaciones pueden extraerse del entorno en que realmente se ejecutan. Esta desvinculación facilita el despliegue uniforme de las aplicaciones basadas en ellos con independencia de que el entorno sea un centro de datos privado, la nube pública o el portátil personal de un desarrollador.



    Desarrollo ágil
    Los contenedores permiten que los desarrolladores avancen con mucha más rapidez, ya que se pueden despreocupar de las dependencias y los entornos.

    Operaciones eficientes
    Los contenedores son ligeros y te permiten utilizar los recursos de computación indispensables, ni más ni menos. Así puedes ejecutar tus aplicaciones de manera eficiente.

    Ejecución en cualquier lugar
    Los contenedores se pueden ejecutar casi en cualquier lugar. Donde quieras ejecutar el software, podrás usar contenedores


    En la virtualización tradicional, ya sea en las instalaciones o en la nube, se aprovecha un hipervisor para virtualizar el hardware físico. Luego, cada máquina virtual contiene un sistema operativo invitado, una copia virtual del hardware que el sistema operativo necesita para ejecutarse, junto con una aplicación y sus bibliotecas y dependencias asociadas.
    En lugar de virtualizar el hardware subyacente, los contenedores virtualizan el sistema operativo (normalmente Linux) para que cada contenedor individual contenga solo la aplicación y sus bibliotecas y dependencias. La ausencia del sistema operativo invitado es el motivo por el que los contenedores son tan ligeros y, por lo tanto, rápidos y portátiles.


    Desarrollo de aplicaciones para la nube

    Red de servidores en los que podemos acceder a programas sin necesidad de que estén instalados

    Desarrollo de aplicaciones para la nube

    El desarrollo de aplicaciones para la nube, también conocido como desarrollo de aplicaciones en la nube o desarrollo de software en la nube, se refiere al proceso de crear aplicaciones que se ejecutan en servidores remotos y se entregan a través de internet. Esta tecnología evita que las empresas tengan que encargarse de aprovisionar, configurar o gestionar los recursos y permite que paguen únicamente por los que usen. Aquí hay algunos pasos clave en el proceso de desarrollo de aplicaciones para la nube:


    Proceso de desarrollo de una app para la nube:

    1. Definicion de requisitos: Al igual que en cualquier proyecto de desarrollo de software, es crucial comenzar por comprender los requisitos del cliente o del usuario final. Esto implica definir las características y funcionalidades que la aplicación debe tener, así como cualquier requisito específico relacionado con la nube, como escalabilidad, disponibilidad y seguridad.

    2. Elección de la plataforma de la nube: Existen varias plataformas de computación en la nube disponibles, como Amazon Web Services (AWS), Microsoft Azure, Google Cloud Platform (GCP), entre otras. Es importante seleccionar la plataforma que mejor se adapte a las necesidades del proyecto en términos de costo, escalabilidad, servicios ofrecidos y soporte.

    3. Diseño de la arquitectura: La arquitectura de la aplicación en la nube debe diseñarse teniendo en cuenta los principios de escalabilidad, disponibilidad y rendimiento. Esto puede incluir decisiones sobre la distribución de los componentes de la aplicación, la selección de servicios de la nube para diferentes funciones (almacenamiento, procesamiento, bases de datos, etc.) y la implementación de prácticas de diseño resilientes.

    4. Desarrollo de la aplicación: Durante esta etapa, se escribe el código de la aplicación utilizando las tecnologías y herramientas adecuadas. Esto puede incluir el desarrollo de la lógica de la aplicación, la integración con servicios de la nube y la implementación de medidas de seguridad.

    5. Pruebas:Es fundamental realizar pruebas exhaustivas de la aplicación para garantizar su funcionamiento correcto en el entorno de la nube. Esto puede incluir pruebas de unidad, pruebas de integración, pruebas de carga y estrés, así como pruebas de seguridad.

    6. Implementación y despliegue: Una vez que la aplicación ha sido probada y validada, se implementa en el entorno de producción en la nube. Esto implica configurar los recursos necesarios en la plataforma de la nube y desplegar la aplicación para que esté disponible para los usuarios finales.

    7. Montioreo y mantenimiento: Una vez que la aplicación está en funcionamiento, es importante monitorear su rendimiento y disponibilidad para detectar y solucionar cualquier problema que pueda surgir. Además, es posible que sea necesario realizar actualizaciones periódicas para agregar nuevas características o mejorar el rendimiento y la seguridad de la aplicación.


    El desarrollo de aplicaciones para la nube implica adaptar el proceso de desarrollo de software tradicional para aprovechar las ventajas y capacidades de la computación en la nube, como la escalabilidad, la disponibilidad y la flexibilidad.


    ¿Cómo funciona la nube?

    Las empresas o personas físicas pagan para acceder a un grupo virtual de recursos compartidos, incluidos servicios de computación, almacenamiento y redes, que se encuentran en servidores remotos propiedad de proveedores de servicios y gestionados por ellos.

    Es decir, utiliza una red (normalmente, Internet) para conectar a los usuarios a una plataforma en la nube donde solicitan y acceden a servicios informáticos alquilados. Un servidor central gestiona toda la comunicación entre los dispositivos y los servidores de cliente para facilitar el intercambio de datos. Las funciones de seguridad y privacidad son componentes habituales para mantener la seguridad de esta información.


    Modelos de despliegue

  • Nubes públicas: Las ejecutan proveedores de servicios en la nube externos. Ofrecen recursos de computación, de almacenamiento y de red por Internet, lo que permite a las empresas acceder a recursos bajo demanda compartidos en función de sus requisitos únicos y de sus objetivos empresariales.

  • Nubes privadas: Se crean, gestionan y pertenecen a una sola organización, y se alojan de forma privada en sus propios centros de datos, lo que comúnmente se denomina "on-premise". Proporcionan más control, seguridad y gestión de datos, al mismo tiempo que permiten que los usuarios internos se beneficien de un conjunto compartido de recursos de computación, almacenamiento y redes.

  • Nubes híbridas: Combinan modelos de nube pública y privada con los que las empresas pueden aprovechar los servicios de nube pública y mantener las funciones de cumplimiento y seguridad habituales. en arquitecturas de nube privada.


    • Tipos comunes de aplicaciones que se desarrollan en la nube

    Aplicaciones Web

    Se ejecutan en un navegador web y no requieren que el usuario descargue ni instale software adicional. Pueden ofrecer funcionalidades simples o complejas y son accesibles desde cualquier dispositivo con conexión a internet.


    Aplicaciones Moviles

    Pueden almacenar datos y procesar tareas en servidores remotos en lugar de en el dispositivo del usuario. Esto permite que las aplicaciones móviles accedan a recursos adicionales y se integren con otras aplicaciones y servicios en la nube.


    Aplicaciones Empresariales

    Proporcionan herramientas y servicios para empresas, como recursos de TI, gestión de recursos humanos, contabilidad, CRM (Customer Relationship Management), ERP (Enterprise Resource Planning), entre otros. Estas aplicaciones permiten a las empresas escalar y gestionar sus operaciones de manera más eficiente.


    Aplicaciones de Almacenamiento y Colaboración

    Permiten a los usuarios almacenar, compartir y sincronizar archivos y datos en servidores remotos. Esto facilita la colaboración entre equipos distribuidos geográficamente y el acceso a archivos desde cualquier lugar y dispositivo.


    Aplicaciones de Inteligencia Artificial y Aprendizaje Automático

    Utilizan recursos en la nube para procesar grandes volúmenes de datos y entrenar modelos de aprendizaje automático e inteligencia artificial. Pueden ofrecer funcionalidades como análisis de datos, reconocimiento de patrones, recomendaciones personalizadas, procesamiento de lenguaje natural, etc.


    Aplicaciones IoT (Internet de las cosas)

    Recopilan datos de dispositivos conectados a internet, como sensores, dispositivos domésticos inteligentes, vehículos, etc., y los procesan en la nube para análisis, monitoreo y control remoto. Esto permite crear soluciones inteligentes y conectadas para una variedad de industrias y aplicaciones.


    Aplicaciones de Juegos

    Utilizan recursos de procesamiento y almacenamiento remotos para ejecutar juegos complejos y gráficamente intensivos en dispositivos con recursos limitados, como teléfonos móviles o dispositivos de transmisión de juegos.


    Aplicaciones de Salud y Bienestar

    Proporcionan servicios relacionados con la salud y el bienestar, como seguimiento de la actividad física, monitoreo de la salud, telemedicina, gestión de registros médicos electrónicos, etc. La computación en la nube permite almacenar y procesar grandes volúmenes de datos de salud de manera segura y escalable.

    ASP.NET es un marco web de código abierto, creado por Microsoft, para crear servicios y aplicaciones web modernos con .NET.
    ASP.NET es multiplataforma y se ejecuta en Windows, Linux, macOS, and Docker.

    ASP.NET

    ¿Qué es ASP.NET Core?

    ASP.NET Core es un marco de desarrollo de aplicaciones web de código abierto y multiplataforma desarrollado por Microsoft. Es una evolución de ASP.NET, el marco de desarrollo web de Microsoft que ha sido utilizado durante muchos años para construir aplicaciones web en la plataforma Windows. ASP.NET Core fue diseñado para ser modular, ligero y de alto rendimiento, lo que lo hace adecuado para desarrollar aplicaciones web modernas y escalables.


    ASP.NET amplía la plataforma .NET con herramientas y bibliotecas específicas para compilar aplicaciones web. Estas son algunas cosas que ASP.NET agrega a la plataforma .NET:


    1. Marco base para procesar solicitudes web en C# o F#
    2. Sintaxis de plantillas de páginas web, conocida como Razor, para crear páginas web dinámicas con C#
    3. Bibliotecas para modelos web comunes, como El controlador de vista de modelos (MVC)
    4. Sistema de autenticación que incluye bibliotecas, una base de datos y páginas de plantilla para controlar inicios de sesión, incluida la autenticación multifactor y la autenticación externa con Google, X, y más
    5. Extensiones del editor para proporcionar resaltado de sintaxis, finalización de código y otras funciones específicas para desarrollar páginas web


    ¿Qué es Web Forms?

    Los formularios Web Forms son páginas que los usuarios solicitan mediante su explorador. Estas páginas se pueden escribir con una combinación de HTML, script de cliente, controles de servidor y código de servidor. Cuando los usuarios solicitan una página, el marco se compila y ejecuta en el servidor y luego el marco genera el marcado HTML que el explorador puede representar. Una página de ASP.NET Web Forms presenta información al usuario en cualquier dispositivo cliente o explorador.

  • Se basa en la tecnología de Microsoft ASP.NET, en la que el código que se ejecuta en el servidor genera dinámicamente la salida de la página web en el explorador o el dispositivo cliente.
  • Es compatible con cualquier explorador o dispositivo móvil. Una página web de ASP.NET representa automáticamente el código HTML compatible con el explorador correcto para características como estilos, diseño, etc.
  • Se admite con cualquier lenguaje compatible con Common Language Runtime de .NET, como Microsoft Visual Basic y Microsoft Visual C#
  • Se basa en Microsoft .NET Framework. Esto proporciona todas las ventajas del marco, incluido un entorno administrado, seguridad de tipos y herencia.
  • Es flexible porque puede agregar en ellos controles creados por el usuario y de terceros.



    • Windows Communication Foundation (WCF)

    Marco para crear aplicaciones orientadas a servicios. Con WCF, es posible enviar datos como mensajes asincrónicos de un punto de conexión de servicio a otro. Un extremo de servicio puede formar parte de un servicio disponible continuamente hospedado por IIS, o puede ser un servicio hospedado en una aplicación. Un extremo puede ser un cliente de un servicio que solicita datos de un extremo de servicio. Los mensajes pueden ser tan simples como un carácter o una palabra que se envía como XML, o tan complejos como una secuencia de datos binarios.

    Algunas caracteristicas
    • Orientación a servicios: La arquitectura orientada a servicios es el uso de servicios web para enviar y recibir datos. Los servicios tienen la ventaja general de estar débilmente acoplados entre una aplicación y otra en lugar de incluidos en el código. Una relación de acoplamiento débil implica que cualquier cliente creado en cualquier plataforma puede conectar con cualquier servicio siempre y cuando se cumplan los contratos esenciales.
    • Interoperabilidad: Implementa estándares del sector modernos para la interoperabilidad de los servicios web.
    • Varios patrones de mensaje: El más común es el de solicitud/respuesta, en que un extremo solicita datos de otro extremo. y el otro extremo responde. Existen otros patrones, como un mensaje unidireccional, en que un único extremo envía un mensaje sin esperar ninguna respuesta. Un patrón más complejo es el patrón de intercambio dúplex donde dos extremos establecen una conexión y envían datos hacia delante y hacia atrás, similar a un programa de mensajería instantánea.
    • Metadatos de servicios: Admite la publicación de metadatos de servicios con los formatos especificados en los estándares de la industria, como WSDL, Esquema XML y WS-Policy. Estos metadatos pueden utilizarse para generar y configurar clientes de forma automática para el acceso a los servicios de WCF.
    • Contratos de datos: Dado que WCF se compila con NET Framework, también incluye métodos con código sencillo para proporcionar los contratos que quiere aplicar. Uno de los tipos de contrato universales es el contrato de datos.
    • Seguridad: Es posible cifrar los mensajes para proteger la privacidad, así como obligar a los usuarios a que se autentiquen antes de permitirles recibir mensajes. La seguridad puede implementarse utilizando estándares conocidos como SSL o WS-SecureConversation.
    • Transactions: WCF también admite las transacciones que usan uno de los tres modelos de transacción: las transacciones WS-Atomic, las API del espacio de nombres System.Transactions y el Coordinador de transacciones distribuidas de Microsoft.


    Modelo Vista Controlador MVC

    El patrón arquitectónico de Modelo-Vista-Controlador (MVC) separa una aplicación en tres componentes principales: el modelo, la vista y el controlador. El marco de MVC de ASP.NET proporciona una alternativa al patrón de ASP.NET Web Forms para crear aplicaciones web basadas en MVC. El marco de ASP.NET MVC es un marco de presentación ligero y muy fácil de probar que (como las aplicaciones basadas en Web Forms) se integra con las características de ASP.NET existentes, tales como páginas maestras y la autenticación basada en pertenencia. El marco de MVC se define en el espacio de nombres de System.Web.Mvc y es una parte admitida fundamental del espacio de nombres de System.Web.


    El marco de MVC incluye los siguientes componentes:
    1. Modelo: Los objetos modelo son las partes de la aplicación que implementan la lógica del dominio de datos de la aplicación. A menudo, los objetos modelo recuperan y almacenan el estado del modelo en una base de datos.
    2. Vista: Las vistas son los componentes que muestra la interfaz de usuario (IU) de la aplicación. Normalmente, esta interfaz de usuario se crea a partir de los datos del modelo.
    3. Controlador: Los controladores son los componentes que controlan la interacción del usuario, trabajan con el modelo y, en último término, seleccionan una vista para representar la interfaz de usuario.


    Blazor y Razor

    Blazor

    Blazor es un marco web de front-end de .NET que admite la representación del lado servidor y la interactividad del cliente en un único modelo de programación:
    • Cree interfaces de usuario interactivas enriquecidas con C#.
    • Comparta la lógica de aplicación del lado cliente y servidor escrita con .NET.
    • Represente la interfaz de usuario como HTML y CSS para la compatibilidad con todos los exploradores, incluidos los móviles.
    • Cree aplicaciones híbridas de escritorio y móviles con .NET y Blazor.

    Razor

    Razor es una sintaxis de marcado que permite incrustar código basado en servidor (Visual Basic y C#) en páginas web. El código basado en servidor puede crear contenido web dinámico sobre la marcha, mientras que una página web se escribe en el navegador. Cuando se llama a una página web, el servidor ejecuta el comando código basado en servidor dentro de la página antes de devolver la página al navegador. Al ejecutarse en el servidor, el código puede realizar tareas complejas, como acceder a Bases.

    Signal R

    SignalR de ASP.NET es una biblioteca para los desarrolladores de ASP.NET que simplifica el proceso de agregar funcionalidad web en tiempo real a las aplicaciones. La funcionalidad web en tiempo real es la capacidad de disponer de contenidos de inserción de código de servidor en clientes conectados al instante, a medida que van estando disponibles, en lugar de que el servidor tenga que esperar que un cliente solicite nuevos datos.


    Tipos de aplicaciones web con .NET

    En el desarrollo web con .NET, existen varios tipos de aplicaciones que se adaptan a diferentes necesidades y contextos.
    1. Aplicaciones Web ASP.NET: Las aplicaciones web ASP.NET son aquellas que siguen el modelo de desarrollo tradicional basado en servidores web, utilizan tecnologías como ASP.NET Web Forms o ASP.NET MVC para crear páginas web dinámicas que se generan en el servidor y se envían al cliente.

    2. Aplicaciones Web de una sola pagina: Las aplicaciones web de una sola página (Single Page Applications, SPA) son aquellas que cargan una sola página HTML y actualizan dinámicamente el contenido según la interacción del usuario, sin necesidad de recargar la página completa.

    3. Aplicaciones Web de tiempo real: Las aplicaciones web de tiempo real son aquellas que permiten la comunicación bidireccional entre el cliente y el servidor, lo que permite actualizaciones en tiempo real sin necesidad de que el usuario actualice la página. Estas aplicaciones son ideales para aplicaciones que requieren notificaciones instantáneas, chats en línea, juegos multijugador, entre otros.

    4. Servicios Web, Microservicios y Web APIs: Los servicios web, microservicios y Web APIs son componentes clave en la arquitectura de aplicaciones web modernas, permiten la comunicación entre diferentes partes de una aplicación o entre aplicaciones diferentes a través de la web, estos servicios suelen seguir estándares como REST o SOAP para definir cómo se comunican.
